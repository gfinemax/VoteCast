# VoteCast 사용자 매뉴얼 (User Manual)

## 🔐 시스템 접속 및 보안
*   **관리자 로그인**: 시스템 보안을 위해 승인된 구글 계정으로만 접속이 가능합니다. (`/login`)
*   **인증 상태 확인**: 헤더 우측의 사용자 버튼을 눌러 내 이메일을 확인하거나 로그아웃할 수 있습니다.


## 📌 안건 설명 자료 (PPT) 설정 가이드

VoteCast는 두 가지 방식으로 안건 설명 자료를 송출할 수 있습니다.
1. **마스터 파일 방식 (추천)**: 하나의 PPT 파일을 등록하고, 각 안건별로 **페이지 번호**만 지정합니다.
2. **개별 파일 방식**: 안건마다 서로 다른 파일을 등록합니다.

---

### 1️⃣ 방식 A. 마스터 PPT (통합 파일) 사용하기 [추천]

가장 관리가 쉽고 끊김 없는 방식입니다.

#### 1. 마스터 파일 등록
1.  **관리자 화면** (`/admin`) 상단의 **[PPT 설정]** 버튼을 누릅니다. (구현 예정)
2.  **통합 자료(Master Presentation)**에 **웹 링크(Google Slides)** 또는 **PDF 파일**을 등록합니다.
3.  **저장**을 누릅니다.

#### 2. 안건별 페이지 지정
1.  안건 목록에서 각 안건의 **수정(연필 아이콘)** 버튼을 누릅니다.
2.  **[시작 페이지]** 칸에 해당 안건이 시작되는 페이지 번호를 숫자(예: `3`)로 입력합니다.
3.  **PDF의 경우**: 실제 PDF 뷰어 기준 페이지 번호입니다.
4.  **Google Slides의 경우**: 슬라이드 번호입니다.

#### 3. 송출
*   관리자가 안건을 클릭할 때마다 송출 화면(Projector)이 **자동으로 해당 페이지로 점프**합니다.

---

### 2️⃣ 방식 B. 개별 파일 업로드

안건별로 자료 형식이 다르거나 별도의 영상/문서를 보여줘야 할 때 사용합니다.

1.  안건 수정 화면에서 **[개별 자료 설정]**을 켭니다.
2.  직접 **웹 링크**를 입력하거나 **파일**을 업로드합니다.
    - **[NEW]** 파일 업로드 시 사용자가 올린 **원본 파일명**이 UI에 표시되어 관리하기가 더 편리해졌습니다.
3.  이 경우 마스터 파일 설정보다 **개별 설정이 우선** 적용됩니다.

---

### 3️⃣ 송출 화면(Projector) 최적화

화면 전환 시 끊김 없는 경험을 위해 다음과 같은 기능이 적용되어 있습니다.

*   **부드러운 화면 전환**: '성원보고 ↔ 안건설명 ↔ 투표결과' 모드 변경 시 화면이 깜빡이지 않고 부드럽게 교차(Cross-fade)되며 전환됩니다.
*   **즉각적인 자료 로딩**: 안건 설명용 PDF가 백그라운드에서 미리 로드되어 있어, 모드 전환 시 대기 시간 없이 즉시 슬라이드가 나타납니다.
*   **송출창 상태 표시 (ON AIR)**:
    - **송출창 버튼**을 누를 때마다 송출 화면을 **여러 개 추가로 열 수 있습니다**.
    - 송출창이 하나 이상 열려 있으면 버튼이 **[송출창 추가 열기 (개수)]** 형태로 바뀌고 하얀색 점이 깜빡입니다.
    - 각 송출창은 창 우측상단의 **닫기(X)** 버튼으로 개별 종료할 수 있습니다.
    - 관리 화면에서는 필요 시 **[송출창 전체 닫기]** 버튼으로 모든 송출창을 한 번에 종료할 수 있습니다.
*   **풀스크린 최적화**: PDF 뷰어의 도구 모음과 스크롤바가 자동으로 숨겨져 깔끔한 전체 화면 발표가 가능합니다.
*   **선포문 자동 확장**: 관리자 패널의 '선포 문구' 입력칸은 내용 길이에 따라 자동으로 높이가 조절되어, 스크롤 없이 전체 내용을 한눈에 확인하며 편집할 수 있습니다.
*   **즉시 출석 및 안건 동기화 [UPDATED]**: 입구에서 스마트폰으로 체크인/취소하거나 관리자 화면에서 안건 내용을 수정하면, 열려 있는 모든 화면에 1초 이내로 즉시 반영됩니다. 새로고침이 필요 없습니다. 스마트폰 화면을 잠갔다가 켜도 자동으로 최신 데이터가 동기화됩니다.

### 4️⃣ 투표 및 의결 확정 가이드 (Vote Snapshot)

안건 의결 후 성원 변동에 영향을 받지 않고 결과를 고정하기 위한 기능입니다.

1.  **실시간 투표 집계**:
    - **[자동계산 ON]** 버튼(헤더 오른쪽)을 켜면 찬성/반대/기권 입력 시 나머지가 자동으로 계산됩니다.
    - 분할 입력 방식(서면+현장 구분) 화면에서 서면과 현장 표를 각각 입력하면 **총 합계**가 자동으로 표시됩니다.
    - 하단의 '결과 일치 (검증됨)' 배지가 녹색인지 확인합니다.

2.  **의결 확정 (🔒 버튼)**:
    - 결과 발표 준비가 되면 **[의결 확정]** 버튼을 누릅니다.
    - 이때의 성원과 투표 결과가 시스템에 **저장(박제)**됩니다.
    - 이후 체크인/아웃으로 성원이 변경되어도, 이 안건의 결과 화면은 변하지 않습니다.

3.  **확정 취소 (🔓 버튼)**:
    - 테스트였거나 입력 실수가 있었을 경우 **[확정 취소]** 버튼을 누르면 초기화됩니다.
    - 다시 실시간 성원 데이터와 연동되는 상태로 돌아갑니다.

---

### 5️⃣ 검산 및 확인서 활용 (Tally Audit) [NEW]

전체 투표 결과를 검산하고, 데이터 오류(중복 투표 등)를 찾아내어 최종 확인서를 작성하는 기능입니다.

1.  **접속 방법**:
    - 관리자 메인 화면 상단의 **[검산 및 확인서]** 버튼을 클릭합니다.
2.  **안건별 검산 현황**:
    - 화면 상단에서 출석 인정 인원과 실제 투표(찬성+반대+기권) 합계가 일치하는지 확인합니다.
    - 불일치하는 안건은 **빨간색(Danger)**으로 표시되며, 원인을 파악해야 합니다.
3.  **데이터 무결성 체크**:
    - 우측의 **'검산 이슈'** 목록을 확인합니다.
    - **중복 투표**: 직접 참석자가 우편 투표도 한 경우 등을 자동으로 찾아냅니다.
    - **입력 누락**: 대리인 성명이 없거나, 서면결의서 필수 기재 사항이 빠진 경우를 표시합니다.
4.  **수기 검산 및 확정**:
    - **[집계 소스]**를 '검산표 확인 집계' 또는 '관리자 수기 확정'으로 변경하여, 프로그램 자동 계산 외에 실물 투표 용지와 대조한 최종 숫자를 직접 관리할 수 있습니다.
5.  **확인서 출력**:
    - 모든 검산이 완료되면 해당 화면을 보고 최종 의결 결과 확인서를 작성하거나 리포트로 활용합니다.

### 💡 팁 (Google Slides 활용)
Google Slides를 사용하면 수정 사항이 즉시 반영되어 매우 편리합니다.
1.  구글 슬라이드에서 **[파일] > [공유] > [웹에 게시]**를 선택합니다.
2.  생성된 링크를 VoteCast에 등록합니다.
3.  슬라이드 내용이 바뀌어도 링크는 그대로이므로, VoteCast 설정을 바꿀 필요가 없습니다.
